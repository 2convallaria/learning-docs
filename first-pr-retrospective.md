## 2. 我学到了什么（流程方面）
1. 关键步骤1：首先要fork一个副本到自己的仓库
2. 关键步骤2：在自己的仓库中找到对应的副本并且进行编辑
3. 关键步骤3：提交修改，填写修改的主题和对应的描述
4. 关键步骤4：选择创建一个新的分支（没有权限），在新分支中修改提交pr
## 3. 如果重来一次，我会如何优化
- 我会：尽量一遍完成自己需要修改的部分，让修改的过程显得简洁明了
## 4. 这个经验对未来贡献的启发
- 启发：真正意义上的第一次走完了一个完整流程的pr,
  这让我在最开始树立了很强的自信心，同时也让我意识到我不是旁观者，而是参与者。
